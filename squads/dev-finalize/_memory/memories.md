## Proibições Explícitas

- **Nunca usar termos técnicos em documentos para o cliente.** Release notes voltadas ao cliente devem usar linguagem de negócio e resultado. Sem: nomes de tecnologias (PostgreSQL, Vue.js, NestJS, MikroORM, Docker, etc.), jargão técnico (deploy, branch, migration, endpoint, CRUD, ORM, JWT). Usar: o que o usuário vai perceber de diferente. Ex: "Agora é possível preencher os dados da empresa automaticamente ao digitar o CNPJ" em vez de "Novo endpoint GET /api/cnpj/:cnpj integrado ao CnpjModule via BrasilAPI".
- O CHANGELOG interno (para a equipe de dev) pode e deve ter os detalhes técnicos. A distinção é: CHANGELOG = interno; release notes para cliente = linguagem leiga.
