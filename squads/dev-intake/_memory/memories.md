## Proibições Explícitas

- **Nunca usar termos técnicos em documentos para o cliente/solicitante.** Isso inclui `perguntas-ao-solicitante.md` e qualquer outro documento voltado para fora da equipe de dev. Sem: nomes de tecnologias (MySQL, Vue.js, NestJS, etc.), infra (Docker, CI/CD, deploy, branch), siglas técnicas (CRUD, ORM, JWT, API, endpoint). Usar linguagem de negócio e resultado. Ex: "preenchimento automático de dados" em vez de "integração com API da Receita Federal via endpoint REST".
