# Step 3 — Checkpoint: Revisão do Conteúdo

## Objetivo

Apresentar o rascunho do conteúdo gerado para revisão antes de montar o HTML.

## Apresentação

Exibir para o usuário o conteúdo redigido para cada feature, em linguagem de negócio, de forma clara.

## Perguntas ao Usuário

Use AskUserQuestion:

**Pergunta:** O conteúdo das release notes está correto?
- **Aprovar e gerar HTML** — Prosseguir para geração do documento HTML
- **Editar manualmente** — Usuário vai corrigir o rascunho antes de continuar
- **Cancelar** — Encerrar sem gerar o HTML

## Se "Editar manualmente"

Orientar o usuário a editar diretamente o conteúdo nos briefings ou a informar as correções em texto.
Após confirmação de que editou, retornar ao Step 2 para re-leitura.

## Se "Aprovar e gerar HTML"

Seguir para Step 4.
